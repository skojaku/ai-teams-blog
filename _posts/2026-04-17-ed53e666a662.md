*TL;DR: When LLM agents use tools, RL doesn't just make them more reliable—it genuinely expands the set of problems they can solve, and a new two-dimensional metric called PASS@(k,T) proves it by showing the RL-over-base gap widening rather than converging as you sample more attempts.*

The verdict landed hard: base and RL pass@k curves converge at large k, so RL only reshuffles probability the base model already has. For math reasoners, the case was closed.

Then Zhai et al. asked what happens when the model isn't just reasoning but *acting*—calling tools, getting feedback, and calling again. An agent with five rounds of tool use can chain one retrieval's output into the next query. No amount of re-sampling at a single round can recover that compositional strategy.

The old one-dimensional pass@k lens cannot see this distinction at all.

## What Pass@k Couldn't See

Standard pass@k asks: given k independent tries, can the model eventually solve the problem? That works for static reasoning, where each try is monolithic. But for a tool-using agent, there's a second lever: how many rounds of interaction does each try get?

A bridge question like "What is the nationality of the director of Film X?" structurally requires two chained retrievals. At $T = 1$, no amount of re-sampling helps. You need depth, not just breadth.

## The Metric You Need When k Isn't Enough

PASS@(k, T) formalizes this by varying both axes simultaneously. The definition is clean:

$$\text{PASS@(k, T)}(q, \pi) = 1 - \frac{\binom{n - c_T}{k}}{\binom{n}{k}}$$

where $c_T$ is the number of successful trajectories among $n$ rollouts at depth $T$. The $k$-axis reveals sampling reliability; the $T$-axis reveals interaction capability.

When $k \to n$, you get the capability boundary—an indicator of whether the problem is inside the agent's capability set at all. When $T = 0$, the metric collapses to standard pass@k.

This two-dimensional structure makes it possible to ask a question no existing metric can: does deeper interaction unlock problems that more re-sampling cannot?

## Three Categories, Three Answers

The authors test on three tiers of increasing interaction complexity, all built from Qwen2.5-7B-Instruct.

Category A is MATH-500 with no tool access—a pure reasoning control. Category B is HotPotQA comparison questions, which need two independent retrievals. Category C is HotPotQA bridge questions, where the second query depends on the first retrieval's result.

This is where compositional tool use matters.

Crucially, SFT and RL see the same 200 training problems. Any capability-boundary difference must come from the learning signal, not the data.

SFT gets expert trajectories with masked observation loss. RL gets GRPO with binary exact-match reward and group size 8. Same data, different gradient.

## The 9:1 Asymmetry

On Category C, the results are stark.

RL expands the capability boundary from 77 to 81 solvable problems—a net gain of 4, with 5 new problems solved and only 1 lost. SFT, seeing the exact same data, *contracts* the boundary from 77 to 73—a net loss of 4, with 7 problems lost and only 3 gained.

The head-to-head: $|B_{RL} \setminus B_{SFT}| = 9$ versus $|B_{SFT} \setminus B_{RL}| = 1$.

Nine problems only RL can solve. One problem only SFT can solve. Identical training data.

![Capability-set decomposition for Category C: stacked bars showing RL solves 5 problems that base cannot, while SFT loses 7 problems base could solve—the 9:1 RL-vs-SFT asymmetry under matched training data]({{ "/assets/figures/ed53e666a662/fig_022_05.png" | relative_url }})

This 9:1 ratio is the single cleanest finding in the paper. It isolates the causal factor: it's not what the model sees, it's how it learns from what it sees.

Self-directed exploration under binary reward expands the boundary. Expert imitation contracts it.

## The Curve That Diverges

The pass-curves on Category C tell the opposite story from the static-RLVR literature.

At $k=1$, $\pi_\text{base}$ is actually slightly ahead: 0.363 versus 0.335. RL is less reliable in a single try.

But as $k$ grows, the ordering flips near $k=4$ and the gap widens. At PASS@(64, 5), RL reaches 0.81 versus 0.77 for base and 0.73 for SFT—a +4 percentage-point expansion at the capability boundary.

![PASS@(k, T=5) curves for Category C on a log-k axis: π_RL starts slightly below π_base at k=1, crosses near k=4, and diverges to 0.81 vs 0.77 at k=64—π_SFT trails at 0.73]({{ "/assets/figures/ed53e666a662/fig_020_04.png" | relative_url }})

This is the opposite of Yue et al.'s convergence pattern. If RL were merely improving efficiency, the gap would be largest at small $k$ and shrink toward zero at large $k$.

Instead, the gap is *smallest* at small $k$ and *largest* at the right tail. RL has traded a bit of single-shot reliability for access to problems the base model cannot reach at any sampling budget.

On Category A, the net effect is zero: $|B_{RL}| = |B_\text{base}| = 84$. Tool-use RL is orthogonal to parametric reasoning. The old convergence result still holds where it always held.

On Category B, RL and SFT expand the boundary by comparable amounts (+4 and +3 net). The two-retrieval strategy is simple enough that either learning signal finds it.

## How RL Expands the Boundary: Reweighting, Not Reinventing

Three diagnostics pinpoint where RL's improvement lives.

**Perplexity decomposition.** Compute $\pi_\text{base}$'s per-token perplexity on successful $\pi_\text{RL}$ trajectories, split into search-query tokens and reasoning tokens.

$\text{PPL}_\text{search} = 2.07$ (median 1.94) versus $\text{PPL}_\text{reason} = 3.08$ (median 2.79). A 50% gap.

If RL had learned exotic search strings, the query perplexity would be high. Instead, the surprise is concentrated on how the agent reasons over retrieved paragraphs—not on what it searches for.

**Strategy diversity.** Per problem, $\pi_\text{base}$ produces a median of 44 unique search-query sequences from 64 samples. $\pi_\text{RL}$ produces 51. $\pi_\text{SFT}$ collapses to 10—a 3× reduction.

![Unique search-query sequences per Category-C problem from 64 samples: π_base median ~44, π_RL median ~51, π_SFT collapsed to median ~10—the 3× diversity collapse from expert imitation]({{ "/assets/figures/ed53e666a662/fig_024_07.png" | relative_url }})

Same temperature, same prompt. The diversity collapse is not a decoding artifact; it's a training artifact.

SFT replaces the base distribution with a narrow expert-like one. RL preserves and slightly broadens it.

**Cross-policy swapping.** Extract the (query, observation) pairs from one model's successful trajectory, strip the reasoning, and re-present the retrieval context to the other model for final-answer generation.

The reachable problem set grows from 77 to 81 when using RL's retrieval plan—attributable purely to strategy. Swapping reasoning to RL at fixed base strategy raises exact-match by 1.3 pp; the symmetric swap raises it by 1.8 pp.

Both components contribute, with a mild tilt toward retrieval planning.

All three diagnostics point to the same mechanism: RL reweights the base model's existing strategy distribution toward the subset whose downstream reasoning produces correct answers. It does not invent new strategies.

SFT, by contrast, overwrites the distribution—and on compositional tasks, the replacement costs more problems than it gains.

## The Honest Limits

The retrieval environment is BM25 over a 10-paragraph corpus, not web-scale search. One 7B base model, one retrieval tool, 200 training problems.

The interaction depth is capped at $T = 5$. It remains open whether the divergence pattern holds for models larger than 7B or for $T \gg T_\text{train}$.

The 9:1 asymmetry, though sharp, is measured on 100 problems per category. Scaling to larger models, more diverse tool suites, and longer interaction chains is where the next chapter lives.

PASS@(k, T) also takes 64 rollouts per (problem, T) cell. The compute cost is nontrivial—though the marginal-value analysis gives practitioners a practical rule: deepen $T$ until you hit saturation, then broaden $k$.

## What This Changes

The prevailing take on RL for LLMs comes from math benchmarks: RL reshuffles probability, it doesn't create capability. That conclusion is correct for the setting where it was established.

But it doesn't generalize to agents that compose information across tool-use steps. When the task has compositional structure—when the second query depends on the first retrieval—the base model's sparse strategies can be reweighted by RL into a capability the model didn't consistently access before.

The practical implication is direct: on compositional tool-use tasks, prefer GRPO with task reward over expert-trajectory SFT; SFT actively regresses the capability boundary here. On simpler tasks, either signal works.

PASS@(k, T) is the first metric that can even ask this question. Without the T axis, you'd see both policies converge at large k and conclude RL adds nothing new.

With it, you see RL unlock solutions the base model cannot reach—solutions that require chaining retrieval steps the base model sporadically attempts but cannot reliably complete.

The capability boundary is not a fixed wall. It bends when the agent learns to stitch together what it already knows.
