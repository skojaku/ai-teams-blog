*TL;DR: Five major LLMs given the same API-discovery task agree on roughly half the results—but disagree sharply on which ones matter most. This paper builds the first benchmark that measures that hidden ranking divergence, and shows it's structured, domain-dependent, and a real safety risk for multi-agent systems.*

## The Same Task, Different Answers

Ask ChatGPT, Claude, Gemini, DeepSeek, and Mistral to find the top 10 weather APIs. They'll overlap on maybe five.

Ask them to rank those ten by relevance and they'll fall apart—especially at the top, where it matters most.

When LLMs operate as autonomous agents that invoke external APIs, we assume that if they retrieve similar tools, they'll prioritize them similarly. They don't. The gap between "we found the same APIs" and "we agree on which one to call first" is where multi-agent coordination quietly breaks.

## Why This Divergence Matters

LLMs are increasingly deployed as team members—not solo chatbots. Multi-agent frameworks like AutoGen wire models together to reason, critique, and coordinate.

The whole premise assumes that when agents face the same task, they'll converge on compatible actions.

Prior work gave us no reason to question it. Toolformer taught LLMs to teach themselves tool use. API-Bank benchmarked how well a single model could call tools. SOLAR, from the same author—Eyhab Al-Masri—measured how one LLM ranks APIs.

None of them asked: *do different models even agree?*

This paper asks that question.

The answer is sobering.

## Building the Divergence Lens

Al-Masri built an automated pipeline on AutoGen that runs five model families—ChatGPT-5, Claude 4.5 Sonnet, Gemini 2.5 Flash, DeepSeek, and Mistral Large 2—through identical API-discovery prompts across 15 domains.

Each model returns a ranked list of 10 APIs per query. That's 750 ranked lists and 14,250 atomic data points.

Then every pair gets hit with complementary metrics:

- **Set-based**: Average Overlap (AO) and Jaccard—do they find the same APIs at all?
- **Rank-sensitive**: Rank-Biased Overlap (RBO) and Kendall's τ—do they order them the same way?
- **Group-level**: Kendall's W and Cronbach's α—does the whole team behave coherently?

Plus volatility scoring and Kemeny–Young consensus distance to capture how much rankings jump around per domain.

The key distinction: *retrieval overlap* (same APIs, any order) versus *rank-order agreement* (same priorities). The gap between those two is where the story lives.

## What They Found

The headline numbers: mean AO ≈ 0.50, Jaccard ≈ 0.36, τ ≈ 0.45. Models share about half their APIs and agree on ordering roughly 45% of the time.

But mean RBO ≈ 0.33. That's the number that should make you pause. RBO weights the top of the list more heavily—these are the APIs an agent would actually execute first. Agreement drops sharply exactly where the stakes are highest.

A three-tier reliability hierarchy emerges:

- **High-stability**: Claude–DeepSeek (AO = 0.58, τ = 0.65), Claude–Mistral (AO = 0.56, τ = 0.62), DeepSeek–Mistral (AO = 0.53, τ = 0.62). These pairs retrieve and rank coherently.
- **Moderate-stability**: Claude–Gemini, DeepSeek–Gemini, Gemini–ChatGPT, DeepSeek–ChatGPT, Gemini–Mistral. Overlap is decent (AO ≈ 0.47–0.56) but rank alignment weakens (τ ≈ 0.39–0.43).
- **Low-stability**: Claude–ChatGPT (AO = 0.46, τ = 0.34) and ChatGPT–Mistral (AO = 0.40, τ = 0.15)—near-random ranking despite some retrieval overlap.

$$\tau = \frac{C - D}{0.5 \, n(n-1)}$$

Where $C$ and $D$ count concordant and discordant pairs. ChatGPT and Mistral are barely distinguishable from coin flips on ordering.

## Domains Make or Break Agreement

The same model pair can look reliable or chaotic depending on the task.

Structured, data-bound domains—Weather, Geocoding, Speech-to-Text, SMS—show cumulative reliability totals of 2.1–2.4 across metrics. Volatility is low (≈ 3–4). Consensus distance stays under 0.6. Weather APIs have predictable parameters; everyone agrees on what matters.

![Kendall's W group-reliability distributions across 15 API domains, with tighter higher values for structured tasks like Weather and wider lower values for open-ended tasks like Sentiment Analysis]({{ "/assets/figures/1fea18ee32f6/fig_008_01.png" | relative_url }})

Open-ended, creative domains—Sentiment Analysis, AI Face Generator—collapse. Cumulative totals drop below 1.0. Volatility spikes above 7. Consensus distance exceeds 0.9. When the task requires subjective interpretation, each model follows its own reasoning path.

The pattern is clear: *external structure constrains divergence*.

## The Gap Isn't Noise—It's Structure

A one-way ANOVA, Kruskal–Wallis, and Levene's tests all return p > 0.4. No significant group-level differences across model pairs.

That doesn't mean the variation is random. It means the divergence falls on a continuous gradient, not into discrete clusters. When you add the domain-level view, the structure sharpens: the same pairs that are coherent on Weather are incoherent on Sentiment Analysis.

The behavior is domain-dependent, not arbitrary. And that means you can predict where coordination will hold and where it will fray. A multi-agent system handling weather lookups? Probably fine. One deciding sentiment-based actions? Needs consensus filtering or selective routing—don't just majority-vote and hope.

## The Safety Angle Nobody Is Checking

Here's the sharp edge: surface-level agreement can mask action-relevant instability.

Two agents retrieve the same API list, so the orchestrator logs "consensus achieved." But one agent ranks the free tier first and the other ranks the authenticated endpoint first. Same tools, different execution path.

Downstream behavior diverges—and in safety-critical contexts, that's not a rounding error. That's a latent coordination failure.

Standard single-agent evaluations miss this entirely. They test whether one model can find and rank APIs correctly. They never test whether two models agree on what "correct" means.

This paper gives practitioners a pre-deployment diagnostic: run the benchmark, check which tier your model pairs fall into, weight confidence by domain structure. Don't deploy a ChatGPT–Mistral pair on sentiment tasks and expect coherent behavior.

## Name the Limits

Five models, 15 tasks, 10 APIs per query. That's a solid initial map, not the whole territory.

The study uses fixed temperature and zero-shot prompting—no few-shot, no multi-turn dialogue, no adversarial paraphrasing. The authors flag all of these as future work.

And the benchmark only covers API discovery. Whether these divergence patterns hold for other tool-use tasks—code generation, database queries, system control—remains open.

## What Changes

Old assumption → new reality: *if agents retrieve the same tools, they'll use them the same way* → *retrieval overlap ≈ 0.50 can coexist with rank correlation ≈ 0.15*. You need to measure both.

Same task, different priorities.
