*TL;DR: Eywa wires domain-specific foundation models (time series, tabular) into LLM agent teams through a bidirectional "Tsaheylu" interface, gaining 6.6% more utility and 30% fewer tokens than language-only agents on scientific tasks.*

## The Language Monopoly

Ask a team of LLM agents to forecast solar flare activity from magnetometer time series, and they'll try to reason over serialized numbers in text. The data loses fidelity before the first inference step.

Every major agentic framework today—Debate, Refine, Mixture-of-Agents—assumes natural language is the universal inter-agent wire format. It is not. Scientific data lives in time series, tables, and domain-specific structures that resist faithful serialization.

This paper changes the wiring.

## Enter Tsaheylu

Eywa—named after the planetary network in *Avatar*—gives non-linguistic foundation models a seat at the agent table. The key abstraction is the **Tsaheylu bond**: a bidirectional interface between an LLM and a domain-specific foundation model.

For each domain $k$, Tsaheylu defines a pair $(\phi_k, \psi_k)$:

- $\phi_k : S \to U_k$ — a **query compiler** that translates the LLM's internal state into a structured invocation for FM $F_k$.
- $\psi_k : O_k \to Z_k$ — a **response adapter** that converts the FM's output back into language-compatible context.

The pipeline:

$$\text{task} \xrightarrow{\text{LLM interpret}} s \xrightarrow{\phi_k} u_k \xrightarrow{F_k(\cdot)} o_k \xrightarrow{\psi_k} z_k \xrightarrow{\text{LLM reason}} \hat{y}$$

The LLM decides *when* to invoke the FM through a control policy $C : S \to \{\text{invoke}, \text{skip}\}$. If the FM has nothing useful to contribute, the agent falls back to pure language reasoning.

This skip branch is not a hack—it is what lets EywaAgent subsume language-only agents as a special case.

![Architecture showing the FM-LLM Tsaheylu bond: a query compiler and response adapter bridge an LLM's reasoning loop to a domain-specific foundation model server]({{ "/assets/figures/02bb108c0bf2/fig_001_01.png" | relative_url }})

The authors prove (Theorem 3) that under the **domain advantage assumption**—an FM beats any language-only model on its native modality—EywaAgent attains strictly lower optimal expected task loss than any language-only agent. The bound is strict, not loose.

## Stack the Specialists

Eywa's architecture stacks in three tiers.

**EywaAgent.** A single coupled FM–LLM unit. The LLM reasons; the FM computes. They talk through Tsaheylu. Each FM runs as an independent MCP server—an HTTP endpoint the EywaAgent calls via structured tool invocations.

**EywaMAS.** Drop EywaAgents into existing multi-agent topologies. Debate graph, refine loop, hierarchy—no redesign needed. Replace some LLM workers with EywaAgents and you get a heterogeneous system. When all agents are LLM-only, it degenerates back to a standard MAS.

**EywaOrchestra.** A conductor LLM that, for each incoming task, dynamically selects which FMs to use, which LLM backbones, and which communication topology from a finite pool. No expert configuration required. The conductor reads the task and instantiates the right team on the fly.

The progression is clear: *fixed single agent* → *fixed multi-agent topology* → *adaptive orchestration*.

## EywaBench

Existing scientific benchmarks tend to cover one domain or one data format. The authors introduce **EywaBench**, spanning nine sub-domains across physical (material, energy, space), life (biology, clinic, drug), and social (economy, business, infrastructure) sciences. It covers natural language, time series, and tabular modalities with a unified utility score $u \in [0, 1]$.

The two domain-specific FMs are **Chronos** (time series forecasting) and **TabPFN** (tabular prediction). Neither provides a native language interface—exactly the kind of model that existing agentic systems cannot use.

## What Happens When Specialists Join

The headline numbers are sharp.

EywaAgent improves average utility by **6.6%** over the single-LLM-agent baseline while cutting token usage by **nearly 30%** and execution time by roughly **10%**. The biggest absolute lifts come from domain-specific data: space tasks jump nearly 9 percentage points; business tasks gain 8.4 pp.

![Bar chart comparing EywaAgent to the single-LLM-agent baseline across utility, token consumption, and execution time, showing EywaAgent achieving higher utility with fewer tokens and lower latency]({{ "/assets/figures/02bb108c0bf2/fig_002_02.png" | relative_url }})

In multi-agent settings, EywaMAS achieves **0.6761** overall utility—outperforming Debate MAS (0.6460), Mixture-of-Agents (0.6273), and X-MAS (0.6188). The gap is consistent across all nine sub-domains.

But here is the result that should reframe how you think about agent teams: **heterogeneous LLM-only methods do not consistently beat strong homogeneous baselines on scientific tasks.** Swapping GPT for Claude inside a debate topology barely moves the needle. Swapping in Chronos for time-series data moves it a lot. Cross-modality heterogeneity matters more than LLM-only heterogeneity.

![Overall utility comparison across multi-agent methods: EywaMAS (0.6761), EywaOrchestra (0.6746), Debate MAS (0.6460), MoA (0.6273), and X-MAS (0.6188)]({{ "/assets/figures/02bb108c0bf2/fig_007_04.png" | relative_url }})

EywaOrchestra adds the final twist. With no expert configuration, it reaches **0.6746** overall utility—essentially matching expert-designed EywaMAS (0.6761)—while using 8,335 tokens versus 11,214 and running in 48 seconds versus 72. The conductor learns to route tasks to the right specialists.

## Honest Limits

The Tsaheylu interface pairs $(\phi_k, \psi_k)$ are hand-crafted per foundation model. Automating or learning these mappings is unaddressed. Today, adding a new FM means writing a new query compiler and response adapter by hand.

EywaBench validates the framework on time-series and tabular modalities only. The authors claim the pattern extends to vision, molecular, and geospatial models—GraphCast, AlphaFold, MACE-MP-0 are all cited as candidates—but none are empirically tested here. The theory says it should work; the experiments do not yet confirm it.

## The Deeper Lesson

Most agentic AI research is obsessed with getting LLMs to talk to each other. This paper asks a different question: what if the strongest team member doesn't speak language at all?

The answer is that you build a neural bond—Tsaheylu—and let specialists do what they are best at while the LLM handles reasoning. The Pareto frontier shifts: better answers, fewer tokens, faster runs.

The plug-and-play pattern via MCP is a practical gift. You do not rewrite your agent framework. You add an endpoint.

The deeper lesson is about heterogeneity itself. The field has been chasing LLM-only diversity—mix GPT, Gemini, Claude in a topology table. On scientific workloads, that diversity is mostly noise. The diversity that matters is *modality-native*: put the time-series model on the time-series task, and let the LLM read what it outputs.

Give the banshee a rider, and the clan gets further.
