*TL;DR: The first benchmark for the Agentic Web—where autonomous content agents gate-keep information—shows that decentralized multi-agent coordination usually trails centralized retrieval, but can beat it on question answering; meanwhile, it concentrates nearly all traffic on a handful of dominant websites.*

## The Web Behind Closed Doors

You ask an agent to find something on the web, and it doesn't browse. It negotiates.

Content providers are walling off raw data behind agent-facing APIs and protocols like MCP and A2A. The web is no longer a linked document collection you can crawl. It's a marketplace of autonomous content agents, each controlling their own domain. And until now, nobody had asked the basic question: can anyone actually find good information this way?

AgentWebBench—from Zhong, Shen, and Xiong at CMU—is the first systematic attempt to answer it.

## From One Agent to One Hundred

Prior benchmarks like WebArena and Mind2Web evaluate a *single* agent navigating websites. But when each site is its own autonomous agent, the problem shifts from navigation to *coordination*. You don't scroll a page. You pick which agents to talk to, ask them questions, and piece together an answer from their responses.

AgentWebBench formalizes this as a two-sided architecture. A user agent receives a query, selects which content agents to consult, and synthesizes their responses. On the other side, 100 content agents each control retrieval within their own domain.

The corpus is 18.4 million documents from ClueWeb22-B, partitioned across those 100 websites. No direct corpus access for the user agent. Everything goes through the content agents.

## Three Ways to Coordinate

The benchmark evaluates three coordination strategies that progressively introduce reasoning.

**ToolE** picks websites by embedding similarity and retrieves through tools. No agent communication at all—just automated retrieval.

**ToolP** upgrades website selection to LLM reasoning while keeping tool-based retrieval. The user agent reads descriptions of all 100 websites and reasons about which ones are relevant.

**Multi-Agent** is the full deal. Both sides reason autonomously. The user agent dynamically selects content agents based on evolving information. Content agents retrieve within their own domains, generate summaries, and return structured responses.

This is peer-to-peer coordination with iterative refinement.

Across all three, the baseline is **Classical**: direct dense retrieval over the entire centralized index, no agents involved.

## Four Tasks, One Revealing Picture

AgentWebBench tests four information-seeking tasks.

**Web search** (354 queries from MS MARCO): return a ranked list of documents. Standard IR metrics like NDCG and Recall.

**Web recommendation** (281 histories from ORBIT): given a user's browsing history, recommend what comes next.

**Question answering** (53 queries from DeepResearchGym): produce a short, precise answer through multi-step retrieval and reasoning.

**Deep research** (331 queries from DeepResearchGym): generate a comprehensive report with long-horizon planning.

These span the spectrum from strict retrieval to open-ended synthesis. The results look very different depending on where you land on that spectrum.

## The Expected Gap—and the Reversal

On web search, decentralized coordination loses. The best Multi-Agent NDCG@3 is Gemini-3 at 44.34, compared to Classical at 52.21.

No surprise: you can't beat direct access when the task is pure retrieval.

But question answering flips the script. Gemini-3 under Multi-Agent reaches 67.92% accuracy. The Classical baseline for the same model is 60.38%. Decentralized coordination wins.

Why? Iterative evidence gathering matches the structure of question answering. The user agent can query content agents, read their summaries, decide it needs more, and query again.

That back-and-forth turns a weakness—mediated access—into a strength.

The reversal isn't universal. Smaller models like Qwen3-4B and Qwen3-30B-A3B still lose to Classical on QA. And even GPT-5, a strong model, trails its own Classical baseline on QA accuracy. Coordination pays off—just not for every model.

Web recommendation, meanwhile, is nearly unsolvable. Almost every method posts near-zero recall. The task demands inferring user intent from sparse browsing history across 100 partitioned domains, and current agents simply can't do it.

## Where Things Break

The failure analysis reveals a sharp split by task type.

On question answering, 68–94% of failures come from the *user* agent side—depending on the model. Large majorities point at planning and synthesis, not retrieval.

On web search with strong models like Gemini-3 and GPT-5, the blame reverses: roughly 58–69% of failures come from the *content* agents.

One illustrative case: a question asks what element Plato compares to the Form of the Good. Both content agents return the correct evidence mentioning "light."

The user agent answers "The Sun" instead.

The content agents did their job. The user agent fumbled the last mile.

Website selection is a clear bottleneck. ToolP (LLM-based selection) consistently outperforms ToolE (embedding-based selection) on web search. Picking sites by reasoning rather than similarity can mean the difference of several NDCG points—up to 8 points at NDCG@3 for Gemini-3.

## Who Gets Found?

Here's the ecosystem-level finding that should worry content providers.

Under Classical retrieval, citations spread across many sources—academic repositories, community forums, niche platforms. Under Multi-Agent coordination, citations slam into a small handful of domains. sciencedirect.com and wikipedia.org vacuum up the traffic.

Decentralized access acts as a filter. The user agent gravitates toward sources it trusts and ignores the rest.

That improves reliability in the short run. It also crushes discoverability for smaller providers.

A shift toward the Agentic Web could further concentrate internet traffic and make it harder for most content providers to be found at all.

## Scaling Helps, But Not Enough

Two clear improvement pathways emerge.

First, **test-time scaling** works. Thinking mode—the extended reasoning before action—consistently improves both task performance and interaction reliability across model sizes. It lets agents simulate and verify action sequences before committing, reducing malformed requests and hallucinations.

Second, **bigger models narrow the gap**. Within the Qwen3 series, performance climbs steadily from 4B through 80B-A3B across all coordination strategies. The stronger the model, the closer Multi-Agent gets to Classical—and on QA, it overtakes it.

But scaling alone won't solve web recommendation's near-zero recall. And it won't redistribute traffic from a few dominant domains back to the long tail.

## What This Can't Tell Us

The benchmark uses only the top 100 websites from ClueWeb22-B's English subset. The real Agentic Web will have far more sites, non-English content, and dynamic updates that a static 2022 snapshot can't capture.

Web recommendation's near-zero results across all methods offer almost no signal about what would help.

The traffic concentration finding is observational, not causal. AgentWebBench can show the pattern, but can't yet tell us whether smarter coordination strategies would alleviate or intensify it.

## One Hundred Gatekeepers

The web is being rebuilt as a constellation of autonomous agents, each guarding their own corner.

AgentWebBench shows that coordination can work—sometimes surprisingly well—but it creates chokepoints. A few sites win. Most vanish from view.

The question isn't just whether agents can find information. It's who gets found at all.
