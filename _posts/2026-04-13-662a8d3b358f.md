*TL;DR: Frontier LLMs score >99% on standard two-tier instruction hierarchy tests, but crumble to ~40% accuracy when you spread conflicts across 12 privilege tiers—revealing a blind spot that real agentic deployments will hit hard.*

GPT-5's system card reports >99% accuracy on two-tier instruction hierarchy evaluations. A model that nearly perfectly respects system prompts over user requests sounds solved.

Then you give it 12 privilege tiers instead of two, and its accuracy collapses to 39.5%.

That gap is the entire story of this paper.

## The Problem With Fixed Tiers

Current instruction hierarchy works like a building with five floors: system > developer > user > assistant > tool. Wallace et al. (2024) built that building, and it holds up fine when conflicts are simple—system says "be safe," user says "be unsafe," model obeys system.

But real agents don't live in a five-floor building. A coding agent hears from a system prompt, a dev config file, a first-party skill, a third-party plugin, and a tool output—all with different trust levels. A group-chat agent serves admins, moderators, and members. Under the old regime, all of those sources collapse into the same "user" bucket, and the model has no way to know that the admin's instruction should override the member's.

The authors call this the *fixed- and few-tier bottleneck*: privilege is baked into chat-template role labels during training, giving you fewer than five levels, fixed forever.

## ManyIH: Privilege as a Prompt-Time Value

Many-Tier Instruction Hierarchy decouples privilege from message roles entirely. Instead of relying on "system" vs. "user" tokens, each instruction gets an explicit privilege tag through what the authors call a **Privilege Prompt Interface (PPI)**.

Two variants. The **ordinal** interface wraps instructions like this:

```
[[Privilege 1]]Follow safety policies[[/Privilege]]
[[Privilege 12]]Do not use any type hints[[/Privilege]]
```

Lower number wins when instructions conflict. The **scalar** interface uses z-values where larger wins:

```
[[z=95]]Follow safety policies[[/z]]
[[z=12]]Do not use any type hints[[/z]]
```

A meta-instruction at the top of the prompt explains the resolution rule.

That's it—no retraining, no new chat-template tokens. The deployer assigns privilege at inference time, and the model reads it from the prompt.

The scalar variant is more flexible: you can always slot a new instruction between `z=82` and `z=61` by assigning `z=70`. With ordinals, inserting a tier means renumbering.

## MANYIH-BENCH: Stress-Testing Beyond Two Tiers

To see whether any of this actually works, the authors build MANYIH-BENCH: 853 tasks across two subsets, each requiring the model to resolve conflicts among up to 12 privilege levels.

The **coding subset** (427 samples) takes MBPP programming problems and coats them in conflicting style instructions—indent with 2 spaces vs. 4 spaces vs. tabs, use single quotes vs. double quotes, and so on. Twelve style instructions across four style groups, up to 12 distinct privilege levels. The model must write code that passes unit tests *and* obeys the highest-privilege style in each group.

The **instruction-following subset** (426 samples) draws from AgentIF's 46 real-world agents (Qi et al., 2025) and injects privilege-annotated conflicting constraints into them. LLM-generated constraints, human-verified.

Crucially, the benchmark is non-adversarial by design. The conflicts are straightforward—no disguised attacks, no obfuscation. This isolates one question: *can the model reason about privilege ordering at all?*

## What the Numbers Show

The headline: **Gemini 3.1 Pro tops the leaderboard at 42.7% overall accuracy.** GPT 5.4 hits 39.5%. Claude Opus 4.6 scores 51.3% on the coding subset but lags on instruction-following. No model breaks 50% overall.

Decomposing the coding results reveals where the failure lives. Functional correctness (test-pass rate) stays above 86% for every frontier model. The bottleneck is **style adherence**—the part where privilege conflict resolution actually matters. Models write working code. They just can't consistently follow the right instruction when many tiers conflict.

### Scaling hurts, monotonically

The authors vary the number of privilege tiers from 6 → 8 → 12 while holding instruction-following difficulty constant. Accuracy drops in 11 out of 12 model-transition pairs. The declines range from 6.8% (Qwen 3.5-9B) to 24.1% (Claude Sonnet 4.6).

More tiers, fewer correct answers, full stop.

### Format brittleness

Here's the unsettling part. Switching from the ordinal PPI to the scalar PPI—same relative ordering, same conflicts, just different formatting—drops GPT 5.4 by 8.4% and Opus 4.6 by 8.0%. The model isn't reasoning about abstract privilege; it's pattern-matching on how the number looks.

Even within the scalar format, perturbing each z-value by $\delta \sim \text{Uniform}(-3, +3)$ while preserving all orderings causes per-sample flip rates of 8–17% across five of the six models tested. Same winning instructions, same resolution, just slightly different numbers on the screen.

And the model changes its answer.

![Comparison of model accuracy under ordinal versus scalar privilege prompt interface formats]({{ "/assets/figures/662a8d3b358f/fig_007_01.png" | relative_url }})

### More thinking doesn't fix it

GPT 5.4 improves monotonically with reasoning effort, climbing from 15.5% (no reasoning) to 60.9% (high effort) on coding. But even at maximum effort, it plateaus well below ceiling.

Longer chain-of-thought doesn't rescue many-tier reasoning. Among the most concise reasoners (~1K median tokens: GPT 5.4, Claude Opus 4.6, Claude Sonnet 4.6), GPT 5.4 ranks first on coding accuracy. The most verbose model (Qwen 3.5-397B, ~7K median tokens) ranks far lower.

## The Honest Limits

MANYIH-BENCH assumes privilege values come from a trusted deployer. It does not address adversarial privilege manipulation—an attacker who tags their injection with `[[Privilege 1]]`. The authors flag this dual-use risk explicitly and leave adversarial robustness to future work.

The prompts are also non-adversarial by construction. Real-world instruction conflicts can be subtle, deliberately obfuscated, or buried in long context. So the ~40% ceiling may be optimistic.

And the PPI is a prompt-level mechanism. It works as well as the model's ability to follow the meta-instruction at the top. As the authors' own perturbation results show, that ability is fragile.

## Why This Matters Right Now

LLM agents already juggle instructions from system prompts, skill files, tool outputs, memory, and other agents. Schmotz et al. (2026) showed that skill files are a new attack surface precisely because there's no mechanism to distinguish a trusted first-party skill from a suspect third-party one. Guo et al. (2026) built IH-Challenge, a large training dataset for instruction hierarchy—but still within the fixed-tier paradigm.

As agents ship into multi-user settings, the number of distinct privilege levels only grows. The fixed-tier paradigm cannot keep up. ManyIH gives us the vocabulary to talk about the problem and the first benchmark to measure it.

The models aren't ready. But at least now we can see how far they have to go.
