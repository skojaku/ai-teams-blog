*TL;DR: A lightweight router that picks the right reasoning paradigm per task outperforms any fixed strategy by 2.8pp, recovering up to 37% of the accuracy gap left on the table when you lock in a single approach.*

## You Ship an Agent. It Works. Until It Doesn't.

You ship an LLM agent. It works. Sort of.

The same model crushes one task and fumbles the next. The difference isn't the model. It's the reasoning paradigm wrapped around it.

Chain-of-thought, ReAct, Plan-then-Execute, Reflection, ReCode, or direct prompting. Each changes how the model reasons, calls tools, and spends compute.

The field has positive case studies for each. Nobody has put them head-to-head under the same model, same tools, same evaluation pipeline.

Until now.

## The Experiment

The authors built PARADIGM: a unified framework where six paradigms share the same model interface, evaluation code, and tool stack.

They ran roughly 18,000 completions across four frontier LLMs and ten benchmarks.

The result: no single paradigm wins.

ReAct smashes Direct by 44pp on GAIA where web search is essential. CoT *hurts* GPT-5 by 15pp on HumanEval because step-by-step reasoning disrupts code generation. The best paradigm changes dataset to dataset—ReCode wins on HumanEval, Plan-Execute on SEAL, Direct on NQ.

The oracle tells the real story. Per-task optimal selection outperforms the best fixed choice by 17.1pp on average. That's not noise.

That's headroom.

## Why Your Model Can't Pick Its Own Paradigm

So why not let the model pick its own paradigm? Zero-shot self-routing—and it only works for GPT-5.

GPT-5 self-routes to 67.1% accuracy. Qwen3-Max drops to 42.4%. Qwen3-30B drops to 27.5%. Both fall below their Direct baselines.

Weaker models collapse to a single dominant choice regardless of task requirements. They over-select ReAct at 42–48%, even on knowledge questions where Direct suffices.

The ability to *execute* a paradigm doesn't mean the ability to *choose* one.

Paradigm selection is a distinct meta-reasoning capability.

## Select-Then-Solve

The fix is a learned router.

Before answering each task, a lightweight classifier selects the most suitable paradigm.

The router is simple: task text encoded via `text-embedding-3-small`, concatenated with 22 handcrafted features, and classified with Logistic Regression. One embedding API call per task. A linear prediction. No LLM calls involved.

Labels come from the experimental matrix. For each task, the successful paradigm with the lowest token cost wins. Tasks where all paradigms fail are excluded from training but count as incorrect during evaluation.

![The Select-then-Solve pipeline: a task flows into an embedding-based router that classifies and routes to one of six reasoning paradigms]({{ "/assets/figures/ef4a46efa52c/fig_002_01.png" | relative_url }})

The results:

| Method | GPT-5 | Gemini | Qwen3-Max | Qwen3-30B | Avg |
|---|---|---|---|---|---|
| Direct | 60.3 | 55.5 | 49.8 | 24.9 | 47.6 |
| Best single paradigm | 62.4 | 55.5 | 50.7 | 32.8 | 50.3 |
| **Combined LR router** | **64.2** | **61.0** | **54.6** | **32.8** | **53.1** |
| Oracle | 72.9 | 73.4 | 72.5 | 56.8 | 68.9 |

The router hits 53.1% average accuracy. That's 2.8pp over the best fixed paradigm. 5.5pp over Direct.

More striking: the router achieves higher accuracy than always-ReAct while spending roughly 7.0k tokens per task versus ReAct's 14.0k. Half the compute. Better results.

On GPT-5, the router recovers 31% of the oracle gap. On Gemini, 37%. Not the ceiling—but meaningful progress from a single embedding call.

![Router comparison across four models showing that the embedding-based router consistently outperforms Direct and best fixed paradigm, while self-routing fails for weaker models]({{ "/assets/figures/ef4a46efa52c/fig_017_05.png" | relative_url }})

## What the Router Learns

The router assigns Direct to 47–73% of tasks depending on model capability. GPT-5 gets 66% routed to Direct, consistent with the finding that stronger models need less scaffolding.

Weaker models receive more diverse routing across all six paradigms.

Self-routing shows degenerate biases. GPT-5 over-selects Direct at 65%, achieving 67.1%—which beats Direct but still trails the router. Qwen3-Max and Qwen3-30B over-select ReAct at 42–48%, hurting performance on knowledge-centric tasks.

No model ever selects Reflection under self-routing.

The failure mode is instructive. Weaker models know how to *use* tools. They can't judge *when to*. Paradigm selection requires a calibration that executing the paradigms themselves doesn't build.

## Limits

One implementation per paradigm family. Different prompts, tool interfaces, or search budgets might shift the rankings.

The router trains on roughly 400 labeled tasks per model. Larger datasets and fine-tuned small models would likely improve performance.

This is task-level selection. Within-task adaptive strategies that escalate from Direct to tool-using mid-task are future work.

## The Takeaway

The most effective agents may not be the ones with the most elaborate scaffolds. They're the ones that know when to use structure and when to step aside.

A single embedding call and a linear classifier already outperforms any fixed paradigm choice. The 17.1pp oracle gap is real. Capturing it doesn't require training a new model. It requires a router that learned when to ask for what.
