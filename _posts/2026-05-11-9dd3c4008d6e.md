*TL;DR: Treating the agentic search loop as a POMDP (the CGDP) lets you replace an LLM's lossy implicit memory with a compact predicate-based belief state and a programmatic stopping gate—two plug-and-play interventions that improve multi-hop accuracy by up to 11.4% and cut token costs by up to 39%.*

A coding agent retrieves the same file three times, each time forgetting it already found it.

A RAG pipeline stops after one round because the answer "sounds plausible," even though the evidence is thin.

These aren't edge cases—they're the default failure mode when you ask an LLM to manage its own search state over a long episode.

Kausik, Swaminathan, and Kallus give these failures a formal name: the Context Gathering Decision Process, or CGDP. And from that formalization, two cheap interventions fall out.

## The Problem with Implicit Memory

Every popular agentic harness—ReAct, IRCoT, Iter-RetGen, MemGPT—tracks state the same way: dump every observation into a growing prompt and hope the LLM remembers what matters.

The CGDP framing makes clear why this breaks. As the trajectory grows, the LLM must re-derive "what do I know?" and "what am I still looking for?" at every single step.

The authors map each harness to a four-step loop they call Predicate-Based Adaptive Identification (PBAI): *Stop? → Act → Observe → Update Belief*. Viewed through PBAI, every harness botches at least one step.

IRCoT appends passages without curation—the belief update is implicit and lossy.

ReAct has an explicit Finish action but no mechanism to anchor unresolved questions.

MemGPT lets the LLM manage its own memory, which means stopping is left to poorly calibrated self-assessment.

Iter-RetGen runs a fixed number of rounds—no adaptive stopping at all.

The common root: the LLM acts as policy, belief state, and metacognitive evaluator simultaneously, with no external infrastructure for any of them.

## Two Levers From One Formalism

The CGDP models the agent's problem as a POMDP. The hidden state is the external corpus. Actions are tool calls. Observations are retrieved fragments. The goal: refine a belief state until it isolates task-relevant information.

The optimal agent maximizes expected success while minimizing exploration cost:

$$\arg\max_{\text{Policy}} \mathbb{E}_{(q,c) \sim \mathcal{D}} \left[\text{Success}(q, c, \text{Policy}(q, c)) - \lambda \sum_{t=1}^{T} \text{Cost}(a_t)\right]$$

The authors model LLM behavior within the CGDP as approximate Thompson Sampling—each generation implicitly samples a hypothesis, then queries the environment to test it. PBAI makes this loop explicit.

### The Predicate-Based Belief State

Instead of an unbounded trajectory $s_{t+1} = \text{Truncate}(s_t \oplus \{a_t, o_t\})$, the orchestrator maintains a compact data structure $b_{t+1} = \text{Extract}(b_t, o_t)$ with two components:

- **Facts**: confirmed propositions with evidence from past observations.
- **Open predicates**: unresolved sub-questions the agent still needs to answer.

The extract step runs a lightweight LLM call under 500 tokens. It distills the observation, marks resolved predicates, and appends new sub-questions.

The state is capped at $K \leq 6$ items, so context stays bounded.

The updated $b_{t+1}$ replaces the raw trajectory in the agent's next prompt.

### Stopping Without Self-Assessment

For the stopping decision, the orchestrator doesn't ask the LLM "are you done?" It tracks two signals:

- **Action similarity** (Jaccard overlap between current and recent actions)—catches repetitive looping.
- **Observation novelty** (Unique Passage Rate, or UPR)—catches when the environment is no longer returning new information.

If both stagnation signals fire for $p$ consecutive rounds, the orchestrator halts the loop and forces a final answer from the current belief state. No extra LLM calls. No self-assessment.

## The Lobotomize-then-Replace Experiment

To isolate the effect of each intervention, the authors strip a harness's native memory entirely—lobotomize it so the agent sees only the current observation and the original query—then inject either a structured JSON belief state or a freeform-text belief state.

The lobotomized agents collapse. On MuSiQue, ReAct drops from 29.6% to 12.2%; IRCoT from 14.4% to 9.0%.

Then the belief state restores—and exceeds—the baseline. IRCoT on MuSiQue: 9.0% → 25.9%. ReAct on MuSiQue: 12.2% → 36.8%.

That's up to an 11.4 percentage-point gain over the original baseline.

The freeform-text format beats structured JSON in 10 out of 12 settings. When the extractor is forced into a rigid JSON schema, the LLM produces "no evidence" artifacts for open questions—up to 56% of them.

Freeform text lets the LLM organize information in whatever form preserves the reasoning chain.

The design principle: *the orchestrator should dictate control flow, but the LLM should dictate representation*.

## What the Gate Saves

The exhaustion gate on stateful baselines improves accuracy by up to +6.9 percentage points (SWE-QA-Pro) while cutting wasted tokens.

But the real story is the comparison with LLM self-assessment.

Prompted conservatively, the LLM spirals into infinite loops—costing 159% more tokens than baseline.

Prompted neutrally, it stops prematurely and loses 5% accuracy.

The programmatic gate saves up to 39% of tokens with no accuracy degradation.

![Bar chart comparing token usage and accuracy across stopping strategies: programmatic heuristic saves tokens without accuracy loss, LLM self-assessment either overspends or undershoots]({{ "/assets/figures/9dd3c4008d6e/fig_009_02.png" | relative_url }})

The LLM's self-assessment is not just unreliable—it's actively harmful in both directions.

The Pareto frontier analysis seals the picture. Lobotomization collapses the quality–cost frontier.

Injecting the belief state recovers it. The intervention doesn't shift along the frontier—it fundamentally improves the trade-off.

## Where It Doesn't Reach

All experiments run on a single model: GPT-4o-mini. The framework predicts even larger gains on weaker models where implicit state tracking degrades faster, but that claim is untested.

The CGDP itself is conceptual and empirical, not a theorem with formal bounds. The validation covers retrieval-augmented QA—multi-hop Wikipedia reasoning, conversational memory, code repository search. It does not yet cover code generation, general tool use, or web-scale browsing where the action space is far less structured.

## Why the Framework Matters

Before this paper, fixing an agent's search loop was ad hoc: add a summarizer here, tweak a stopping condition there. The CGDP gives you a shared vocabulary for diagnosing *which* PBAI step is broken—lossy belief update, premature stopping, mode collapse—and a compositional blueprint for fixing it.

The two interventions compose independently. You can add the belief state without the gate, or the gate without the belief state, and each one helps.

The deeper lesson: when you're building an agent harness, stop asking the LLM to do its own bookkeeping.

Offload the state tracking and the stopping decision to code. Let the LLM do what it's good at—reasoning over a clean, compact summary of what it knows.

The search loop doesn't need a smarter agent. It needs a smarter orchestrator.
