## Themes

**Orchestration over parameters.** Every paper today reaches the same conclusion from a different angle: reorganizing how models work together beats retraining them. MAS-Algorithm's five-agent pipeline gains 6.48% on acceptance rate without changing a weight, while LoRA fine-tuning on the same data manages 0.89%. LATTE's dynamic task graphs halve token costs and raise accuracy not by picking a better model, but by giving the team better scaffolding. The model you have, wired right, beats the model you wish you had.

**Adaptive structure beats fixed plans.** Static pipelines assume you know every subtask before execution starts. LATTE shows what happens when you don't: accuracy drops from 79.7% to 57.6% because the team can't grow its graph mid-run. RGAO routes tasks by reading the actual code structure, not just the query text, and cuts misrouting from 30% to 8%. Active learning via EKI picks training tasks based on information gain rather than random sampling, stabilizing the worst quartile of runs. The pattern holds across all five: rigid plans shatter on contact with messy reality.

**The feedback loop is the leverage point.** MAS-Algorithm separates local bugs from global strategy failures through a three-signal loop—PASS, FIX, RETHINK—routing each error type to the agent that can actually fix it. LATTE's DISCOVER, CLAIM, and RELEASE operators let teams adapt mid-execution instead of silently wasting tokens. RGAO verifies budget conservation before dispatching a single agent, not after the tokens burn. Every system in today's batch closes a loop that predecessors left open.

**Baking recurrence into architecture.** Constant-context moves recurring procedures from ever-growing prompts into LoRA adapters and deterministic state trackers, running inference on bounded context. LATTE encodes coordination as graph operators rather than conversational messages. RGAO extracts a five-dimensional complexity vector from the code tree in under a millisecond, instead of re-deriving it at every step. The trend: stop re-reading what you already know, and start carrying it in the structure.

## Editorial

Five papers, one tag, one question: how do you make multi-agent systems not terrible?

Today's batch gives a surprisingly coherent answer. Stop throwing tokens at problems you could structure away. Stop re-reading instructions you've already learned. Stop routing tasks without looking at the code they'll touch. Stop drawing training tasks at random when you can pick the ones that matter.

The genuinely new contribution is RGAO's conservation theorem—proving your budget holds *before* you spend it, not auditing the wreckage after. That's a formal guarantee in a field that usually settles for anecdotes. LATTE's mutable task graph is the other standout: a 22-point accuracy gap between adaptive and static coordination is the kind of number that makes you reconsider every fixed pipeline you've ever built.

What's incremental: MAS-Algorithm's five-agent decomposition is clean, but the individual pieces—retrieval, reasoning, coding, checking—have been floating around the literature for a while. The contribution is the assembly and the three-signal feedback loop, not any single agent. Active learning for communication graphs is a natural idea once you notice that random sampling produces fragile structures; EKI is a solid instantiation but the insight itself was waiting to be had.

What warrants a second look: Constant-context's state trackers. They're hand-specified per task family right now, which limits portability. But the performance jump from prompt-only to SFT-with-tracker—23.6% to 62.2% on WebShop—is enormous. If someone figures out how to learn those trackers rather than write them, that paper becomes the start of something big.

Multi-agent AI is learning what distributed systems figured out decades ago: coordination is the hard part, and the right structure cuts the cost by an order of magnitude before you touch the model weights.
