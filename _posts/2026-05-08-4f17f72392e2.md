*TL;DR: LATTE gives LLM teams a shared, evolving task graph that nearly halves token costs while boosting accuracy — borrowing coordination ideas from distributed computing to fix the structure-versus-flexibility trap in multi-agent systems.*

## When Agents Step on Each Other

Picture five LLM agents trying to debug a signal-processing library together. Two overwrite the same function simultaneously. A third reopens a bug that someone already closed.

By the end, the team has produced 78,000 characters of discarded output — code that never made it into the final product.

That is not a hypothetical. It is what decentralized LLM teams actually do.

## The Structure-Flexibility Trap

Most multi-agent LLM designs fall into one of two camps.

Structured systems — MetaGPT's pipeline, Leader-Worker hierarchies — assign roles and lock sequencing upfront. Efficient when the plan works, brittle when it does not.

In pipelines, a hallucination in the lead cascades unchecked. In hierarchies, workers idle while the lead bottlenecks.

The other camp lets agents self-organize freely — fully decentralized peer teams. Flexibility goes up, but so do the conflicts: overwrites multiply, redundant work piles up, and communication bloats until it consumes more tokens than the actual task.

Mieczkowski and colleagues observed this tension in their earlier work framing LLM teams as distributed systems.

The insight: you do not have to pick a side.

Distributed computing solved this decades ago with task graphs and work-stealing schedulers. LATTE brings that solution to LLM teams.

## The Adaptive Task Graph

The core idea is a shared directed acyclic graph that every agent can see and modify.

Nodes are subtasks. Edges are dependencies — subtask $v$ cannot start until all its predecessors are done. Each node carries a status: `pending → assigned → in_progress → done → verified`.

At any round, the frontier $F_t$ — the set of pending nodes with all dependencies satisfied — tells you how many agents could proceed in parallel.

The graph starts small. A Lead agent initializes $G_0$ with a rough decomposition. From there, the team grows and reshapes it as they learn what the task actually requires.

## Seven Moves

LATTE gives the team seven graph mutation operators, each with explicit preconditions and invariant guarantees:

- **DISCOVER**$(v, \text{deps})$: Add a new subtask node with its dependency edges. Any agent — Lead or Worker — can propose this when new work surfaces mid-execution.
- **ASSIGN**$(v, w)$: The Lead assigns a pending node to a specific Worker.
- **CLAIM**$(v)$: An idle Worker grabs a frontier node for itself — a work-stealing move borrowed from distributed schedulers.
- **COMPLETE**$(v)$: The assigned Worker marks its node as done.
- **RELEASE**$(v)$: The Lead returns a stalled node to pending, freeing it for reassignment.
- **CLOSE**$(v)$: The Lead force-completes a node (for instance, when downstream work is waiting).
- **VERIFY**$(v)$: The Lead spawns a verification check on a high-uncertainty node.

The key split: both Lead and Workers can discover new subtasks, while Workers handle local operations (COMPLETE, CLAIM). The Lead alone controls graph-wide operations (ASSIGN, RELEASE, CLOSE, VERIFY).

All structural updates are serialized — no race conditions on shared state.

## Claim, Don't Wait

The CLAIM operator is where LATTE earns its efficiency gains.

Instead of every assignment going through the Lead, idle Workers pull frontier tasks for themselves. This is the same principle as guided self-scheduling in parallel supercomputing: fast processors grab ready work from a shared queue rather than waiting for a central dispatcher.

The Lead still exists, but it does not bottleneck every micro-decision. Workers self-schedule on the frontier; the Lead steps in for stragglers and verification.

The result: agents are active only 48.7% of rounds, versus 80% in Leader-Worker teams and every round in decentralized ones. Idle compute practically disappears.

## Stragglers Get Caught

A heartbeat mechanism monitors each assigned node. If a Worker stalls for $H$ rounds without completing its subtask, the Lead gets a signal and can invoke RELEASE to put the task back on the frontier.

This is not theoretical. RELEASE fires in 36% of runs.

The impact on tail latency is sharp: average node completion drops from 75.6 seconds (static graphs) to 39.2 seconds. At the 95th percentile, the gap widens to 130 seconds versus 294 seconds — a 2.3× reduction in straggler tail.

## What They Found

Across 300 trials — five team structures, two base models (Claude Sonnet 4-6 and GPT-5.2), three tasks — LATTE Pareto-dominates every baseline on both the accuracy-versus-token-cost and accuracy-versus-wall-clock-time frontiers.

![Pareto frontier showing LATTE dominating all baselines on accuracy versus token cost and wall-clock time]({{ "/assets/figures/4f17f72392e2/fig_006_01.png" | relative_url }})

The headline numbers:

- **Token cost**: Mean normalized cost of 47.5% — nearly half — compared to 86.9% for the next-best method (static graph, $p < 0.01$).
- **Accuracy**: 80% overall, versus 70% Leader-Worker, 74% decentralized, 58% static, 34% MetaGPT.
- **Overwrites**: 4.3 per trial, versus 22.8 for Leader-Worker and 35.4 for decentralized — a 5.3× and 8.2× reduction.
- **Wasted output**: ~5,200 discarded characters per trial, versus ~45,400 (Leader-Worker) and ~78,000 (decentralized).
- **Messages**: 20.4 inter-agent messages per task, versus 31.4 and 34.8.
- **Wall-clock time**: 3.5 minutes aggregate, versus 5.9 for Leader-Worker and 11.5 for MetaGPT.

![Coordination metrics across methods showing LATTE's low overwrites, fewer messages, and less wasted output]({{ "/assets/figures/4f17f72392e2/fig_009_02.png" | relative_url }})

In the Library Extension task, the methods separate sharply: LATTE hits 40%, Leader-Worker manages 23%, MetaGPT collapses to 6%.

## Emergent Verification

One of the quiet details: VERIFY is invoked in only 19% of trials, and almost exclusively in the hard ones.

Teams that complete in 8.1 rounds on average use no verification; teams that take 18.1 rounds trigger it selectively. The Lead does not review everything — it targets high-uncertainty nodes.

That is emergent quality control, and it only becomes visible because LATTE externalizes coordination decisions as graph operations.

## The Honest Limits

Graph initialization adds planning overhead that does not pay off on simple tasks where a single agent would suffice.

The current operator set assumes subtasks with reasonably clear boundaries; open-ended reasoning without natural decomposition points is uncharted territory.

Team size is fixed at $N = 5$ for fair benchmarking against MetaGPT, so scaling behavior remains unknown.

And the probabilistic motivation — treating graph updates as proposal-and-acceptance steps akin to Metropolis-Hastings — is conceptually clean but not yet leveraged for learning better coordination over time.

The coordination metrics LATTE introduces — overwrite rate, concurrent conflicts, wasted characters, idle rounds, straggler tail latency — fill a real evaluation gap.

Most multi-agent LLM papers report accuracy and maybe token count. Nobody was counting how often agents undo each other's work — and that number turns out to be the difference between a team that ships and a team that spins.

The graph does not just coordinate. It makes the coordination legible.
