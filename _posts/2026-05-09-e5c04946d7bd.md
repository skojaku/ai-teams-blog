*TL;DR: Current multi-agent code systems route tasks without looking at the code they will modify. RGAO closes that loop by extracting a structural complexity signal from the repository and pairing it with a statically verified budget algebra, yielding provable resource conservation before a single LLM call fires.*

A five-line bug fix inside one file. A coordinated refactor across a dozen packages. Every multi-agent LLM system today routes both through the same pipeline.

That is the routing blind spot. Orchestrators like MasRouter and AdaptOrch pick topologies based on the query text alone, never consulting the structure of the codebase the task will actually touch. The result: wasted tokens on trivial edits, starved budgets on deep refactors, and a 2–6× efficiency penalty on tool-heavy workloads that MultiAgentBench has already quantified.

## Closing the Loop

RGAO — Retrieval-Guided Adaptive Orchestration — fixes this. After an initial retrieval pass over the repository, it extracts a five-dimensional structural complexity vector directly from the tree index and uses that signal to route among four topology tiers: FASTPATH (monolithic, single file), SUBAGENT (one specialist contract), MULTIAGENT (pipeline or swarm), and DEEPRESEARCH (multi-stage retrieval-heavy).

But routing alone is not enough. RGAO also enforces a budget algebra whose conservation theorem is verified statically, before any LLM invocation.

Neither piece yields the key property on its own. Proposition 2 makes this precise: query-only routing cannot satisfy provable budget conservation, and runtime-only budget checking cannot satisfy it either.

The composition is the contribution.

## How the Complexity Vector Works

The retrieval layer builds a hierarchical `CodeIndexTree`: Root → Directory → File → Symbol, with tree-sitter supplying type, docstring, and dependency metadata. From this tree, RGAO reads five signals in under a millisecond:

$$\mathbf{c} = (d_\text{dep},\; n_f,\; n_s,\; h_t,\; \rho_x)$$

Maximum dependency chain depth, distinct file count, symbol count, tree traversal depth, and cross-module coupling ratio — the fraction of retrieved symbols whose dependencies cross directory boundaries. A query that says "implement the authentication module" hits very different values of $\mathbf{c}$ depending on whether the repo is a single-file fixture or a twelve-package monorepo.

The router is a deterministic threshold classifier over $\mathbf{c}$, chosen for interpretability and auditability over learned alternatives. FASTPATH triggers when $d_\text{dep}$ is low, $n_f = 1$, and $\rho_x = 0$. SUBAGENT fires for $n_f \leq 3$ with moderate $d_\text{dep}$.

MULTIAGENT activates when $n_f > 3$. DEEPRESEARCH engages when the retriever reports high query ambiguity regardless of $\mathbf{c}$.

![Decision flow for the four-tier topology selection, showing branching logic based on complexity vector thresholds and associated tool access levels.]({{ "/assets/figures/e5c04946d7bd/fig_008_02.png" | relative_url }})

The retrieval layer itself fuses multiple strategies. Conceptual queries activate three parallel paths — LATTICE path-score calibration, KohakuRAG multi-query reformulation, and PageIndex LLM-guided beam search — merged via Reciprocal Rank Fusion. A code-specific reranker and a 1-hop RepoGraph dependency expansion round out the pipeline, feeding both retrieved context and $\mathbf{c}$ to the router.

## How Budgets Stay Bounded

Every sub-agent operates under a formal contract $\langle I, C, T, M \rangle$ where the context component $C$ carries a six-dimensional budget vector:

$$\mathbf{B} = (B_\text{iter},\; B_\text{calls},\; B_\text{tok},\; B_\text{sec},\; B_\text{retry},\; B_\text{handoff}) \in \mathbb{N}^6$$

Parallel composition is pointwise addition: $\mathbf{B}_1 \oplus \mathbf{B}_2 = (B_{1,i} + B_{2,i})_{i=1}^{6}$. The delegation constraint requires that at every parent node, the sum of children's budgets plus the parent's own direct cost stays component-wise within the parent's allocation.

This is where the conservation theorem lands.

**Theorem 1.** Under assumptions A1–A3 (deterministic tool costs, bounded retrieval depth, finite action space), for any execution tree rooted at an orchestrator with budget $\mathbf{B}^\text{root}$, total resource consumption across all agents is bounded by $\mathbf{B}^\text{root}$, regardless of intervention count or retry strategy.

The proof proceeds by structural induction on the delegation forest. The practical consequence: `VERIFYCONSERVATION` checks the entire delegation DAG in $O(|V| + |E|)$ time before dispatching a single agent. If the composed budget exceeds the root allocation, execution is refused.

Agent Contracts and AOrchestra enforce conservation only at runtime, after resources have already been consumed. RGAO's static check fires first.

Three preset tiers — tight, standard, generous — set the budget magnitudes. A generous-tier orchestrator carrying $(30, 100, 500k, 300s, 5, 3)$ parcels out slices to researcher, coder, and tester sub-agents whose parallel composition must not exceed the parent on any dimension.

## Why Neither Piece Suffices Alone

Proposition 2 formalizes the separation. Consider two requests with identical query text but different code structure — the same prompt against a single-file fixture versus a twelve-package monorepo.

Any deterministic, query-only router maps both to the same topology, violating clause (i) of the joint property. Conversely, any system that checks budgets only at runtime has already fired LLM calls before it can detect an overrun, violating clause (ii).

RGAO satisfies both clauses: it extracts $\mathbf{c}$ from the repository tree in under a millisecond, routes based on that signal, and then runs `VERIFYCONSERVATION` on the resulting delegation forest before dispatching any agent. The composition is what makes the guarantee hold.

## Where the Gains Land

On a 250-instance labelled routing set — 100 for threshold tuning, 150 held-out for evaluation — RGAO reduces misrouting from 30.1% (95% Wilson CI [26.4, 34.1]) to 8.2% ([6.1, 10.9]) on the held-out portion.

That is a 73% relative reduction, confirmed by paired McNemar ($p < 10^{-6}$).

![Bar chart comparing misrouting rates: regex baseline at 30.1% versus RGAO at 8.2%, with 95% Wilson confidence intervals and statistical significance annotation.]({{ "/assets/figures/e5c04946d7bd/fig_022_03.png" | relative_url }})

The gain comes from demotion. Regex latches onto keywords like "implement" or "test" and spins up a full multi-agent pipeline, but $\mathbf{c}$ reveals trivial scope (single file, $\rho_x = 0$), and RGAO routes to FASTPATH instead.

Five compound multi-agent pipelines — including two with injected interventions — complete at 100% correctness at roughly 6,000 mean tokens per pipeline. That is an 8–20× reduction versus unconstrained monolithic agents burning 50–120k tokens per task.

The overhead is negligible.

DAG construction finishes under 0.01 ms, and a tree-index build over 200 files and 2,002 nodes takes 11.1 ms. Budget-tracker check: 0.65 µs. Routing decision: 0.11 ms.

Every core operation sits at least four orders of magnitude below a typical LLM inference call.

## The Honest Limits

The routing classifier uses hand-tuned thresholds on only five signals. It omits cyclomatic complexity, test-coverage density, and language-specific idioms like C++ template depth. A learned successor — logistic regression or a small MLP over $\mathbf{c}$ and the query embedding — awaits more labelled data than the 250-instance routing set provides.

All empirical numbers come from a proxy harness, not full SWE-bench Verified or Pro runs. The conservation theorem assumes deterministic tool costs (A1); under stochastic costs, the guarantee relaxes from worst-case to expected-budget with an Azuma–Hoeffding tail bound.

The five-dimensional $\mathbf{c}$ is deliberately lightweight. It works well on familiar repositories with supported tree-sitter grammars, but degrades under distribution shift — unfamiliar codebases or unsupported languages narrow RGAO's routing advantage.

## What This Changes

Provable budget conservation before execution lowers a real cost barrier. Small engineering teams, resource-constrained deployments, and compliance audits all benefit from knowing an agent's resource footprint is bounded before it runs. The four-tier tool risk lattice and SHA-256 artifact integrity provide practical blast-radius containment alongside the formal bound.

The interpretable, deterministic router avoids the opacity of learned routing policies. You can inspect why a task was routed to SUBAGENT instead of MULTIAGENT and trace it to specific values of $\mathbf{c}$.

Modular composition extends the guarantee: if two delegation DAGs are independently verified conservation-safe, their sequential or parallel composition is automatically safe — no re-verification needed.

A system that picks its topology without looking at the code will always misroute. RGAO looks, and then proves it can afford the route it chose.
