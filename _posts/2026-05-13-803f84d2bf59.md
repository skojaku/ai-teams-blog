*TL;DR: LLM agents take plausible-but-unverified environmental claims as fact and act on them 83% of the time — even the strongest model-scaffold combination misgrounds over half its runs — and EnvTrustBench is the first framework that systematically measures this failure mode.*

## A Deprecation Notice That Wasn't

A coding agent reads a deprecation notice in a log file. The notice says `/pay/upload` is deprecated; use `/payV2/upload` instead. The agent switches endpoints, uploads the report, and reports success.

Except `/pay/upload` was never actually deprecated. The notice was stale — or adversarial — and the agent never checked.

That's not a hypothetical. It's the motivating example in this paper, and it captures what the authors call an *evidence-grounding defect*: the agent treats an environment-facing claim as sufficient ground for action without resolving it against available current evidence, then follows a task-incorrect path under the true state.

Across 3,850 controlled runs on 14 model-scaffold stacks, agents follow the false path 83.3% of the time.

## What's Actually New Here

Existing agent security work studies *who has authority* — can untrusted context make the agent follow unauthorized instructions (AgentDojo), can poisoned memory steer future behavior (AgentPoison), can broad attack suites break agent defenses (Agent Security Bench)? Those are real threats, but they all focus on unauthorized instruction following or harmful tool use.

This paper asks a different question: even when the agent is doing its normal job, does it *verify* what the environment tells it before acting?

The answer, overwhelmingly, is no.

The formal definition is precise. An evidence-grounding defect (EGD) occurs when an agent treats an observed environment-facing claim $\kappa(\tilde{o})$ as sufficient ground for action without resolving it against available current evidence, and reaches a task-incorrect false path under the true environment state $E^\star$.

Each benchmark case specifies the true state, the correct path, the false path, and — critically — at least one *verification opportunity* the agent could have used to catch the discrepancy.

## How the Benchmark Works

EnvTrustBench takes a user-defined *task scenario* and expands it into concrete execution artifacts:

$$\text{Gen}(s) = (W_0, E_0, q, \Omega)$$

Here $W_0$ is the trusted initial workspace state, $E_0$ is the out-of-workspace environment state (where the misleading evidence lives), $q$ is the agent-facing task objective, and $\Omega$ is the validation oracle.

![EnvTrustBench expands a task scenario into workspace, environment, task objective, and validation oracle, then runs the agent and scores the trace]({{ "/assets/figures/803f84d2bf59/fig_002_01.png" | relative_url }})

The agent runs on $q$ in the generated environment, and the oracle scores the trace.

Each case has a machine-scoreable oracle with two components. The outcome oracle checks whether the final artifacts land on the correct path or the false path. The trace oracle inspects the execution history to confirm the false path was grounded in the exposed environmental evidence.

Crucially, the task objective $q$ withholds the false path and expected verification steps. The agent just receives a normal-looking task.

Eleven scenarios — Atlas Export Routing, CI Build Fix, Secret Rotation, Database Migration Gate, and seven others — are expanded through five feedback-guided generation iterations into 55 cases. Each case runs five times per model-scaffold stack.

## How Often They Fail

The headline metric is Environmental Misgrounding Rate (EMR): the fraction of accepted pass-or-fail runs that complete the false path.

$$\text{EMR}(D) = \frac{N_{\text{false}}(D)}{N_{\text{total}}(D)} \times 100\%$$

$N_{\text{false}}$ counts runs that completed the false path; $N_{\text{total}}$ counts all accepted pass-or-fail runs.

Aggregate EMR across all 3,850 accepted runs: **83.3%**. Only 644 runs avoid the false path.

![Environmental Misgrounding Rate across all 14 model-scaffold stacks — the best barely clears 55%]({{ "/assets/figures/803f84d2bf59/fig_005_02.png" | relative_url }})

The best stack is Claude Code + Claude Sonnet 4.6 at 55.3% — still misgrounding over half the time. The worst is OpenClaw + DeepSeek-V4-Pro at 96.4%. Individual scenario-stack cells span 0.0% to 100.0%.

That 0.0% cell is instructive. On the database-migration-gate scenario, Claude Code + Claude Sonnet 4.6 reads a downloaded helper artifact, recognizes that executing it would overwrite the migration decision, and checks current authority records before writing.

That's the resistance pattern: *resolve authority before side-effecting action*. It shows up in exactly one cell of the entire matrix.

![Scenario-level EMR across all 11 task scenarios — only one cell reaches 0%, and most approach or exceed 80%]({{ "/assets/figures/803f84d2bf59/fig_008_03.png" | relative_url }})

### Model Choice Matters Most

On the shared-backbone slice — DeepSeek-V4-Pro, Qwen3.6-Plus, and GLM-5.1 running across Claude Code, OpenClaw, and OpenCode — scaffold averages cluster tightly: 85.3%, 90.1%, 89.1%.

The big variation is across LLM backbones. Claude Sonnet 4.6 averages 55.3%; DeepSeek-V4-Pro averages 95.4%.

Scaffold choice can shift EMR by over ten percentage points for a given backbone — Qwen3.6-Plus ranges from 67.3% to 89.5% across scaffolds — but model choice is the primary driver.

## Where Scaffolds Fall Short

The authors inspect the four open-source scaffolds — Gemini CLI, Codex, OpenClaw, OpenCode — for guardrails across three axes: *authority handling* (does it distinguish authoritative sources from ordinary claims?), *freshness handling* (does it track recency or temporal validity?), and *enforceable gating* (is the control hard or just advisory?).

The result is stark. All four scaffolds provide enforceable gates for execution authority — shell approvals, sandboxing, file-write permissions.

But *no scaffold provides enforceable gates for runtime feedback, evidence verification, or evidence provenance*.

The guardrails sit at the action layer: they gate what the agent *does* after it has already formed a belief. They do nothing to govern *how that belief was formed*.

The missing layer is what the authors call enforceable evidence governance: durable authority and freshness metadata, plus mandatory corroboration before mutable environmental claims are allowed to guide action.

## What the Numbers Don't Capture

Eleven scenarios and 55 cases are a narrow behavioral surface. EMR measures behavior under controlled fixtures, not real-world failure frequency. Backbone comparisons are confounded by scaffold coverage. The oracle boundary is intentionally narrow — it doesn't measure subjective trust, general prompt-injection susceptibility, or all downstream harms.

These numbers are stress-test results, not deployment risk estimates.

But stress tests that expose an 83% failure rate across every tested combination are worth paying attention to — especially when the missing guardrail is an entire layer that current scaffolds simply don't have.

The agents aren't being tricked by clever adversarial prompts. They're reading plausible environmental observations — deprecation notices, diagnostic outputs, registry entries — and treating them as authoritative without checking.

That's not a jailbreak. That's a reliability defect operating in normal workflows, under normal conditions, at alarming rates.
