*TL;DR: In adversarial partially observable environments, giving LLM agents more reasoning tools and more sub-agents can actively hurt performance — a failure mode the authors call a "deliberation cascade." The cheapest win is deterministic context engineering, not deeper thinking.*

## More Thinking, Worse Results

Add self-questioning, self-critique, and self-improvement tools to every agent in a hierarchy.

Double the token spend.

The reward drops by up to 3.4×.

That is not a misprint. Bogdanov et al. ran 3,475 episodes across six models from five families, burning 283.9 million tokens, to answer a simple question: which design choices actually improve performance per dollar, and which ones just inflate the bill?

The answer reshapes how we build multi-agent systems. What the agent sees matters far more than how deeply it thinks.

## Playing to Not Lose

The testbed is CybORG CAGE-2 — a cyber-defense environment modeled as a Partially Observable Markov Decision Process.

A defender agent protects a 13-host network across 30 steps against a scripted attacker following a multi-stage kill chain.

Every reward is non-positive. The agent is always losing; the only question is how much.

Errors compound across steps and there are no recovery windfalls.

Returns below −150 — roughly the level of a random agent — happen more than 90% of the time when fed raw, unprocessed observations.

The authors use knowledge-free initialization: a one-sentence role instruction and an action reference table.

No topology hints, no attacker model, no benchmark-specific cues.

Performance comes from architecture and runtime context, not prompt engineering.

## Three Levers, One Metric

The study varies three design axes that practitioners routinely compose when building compound agents.

**Context** — six configurations, from raw observation dumps to structured summaries maintained by a deterministic state-tracking layer. The key variable is whether the agent parses noisy process tables itself or receives a compressed `{network_status}` that collapses to a single sentence when everything is healthy.

**Deliberation** — four cumulative levels added to the monolithic Planner: self-questioning, self-critique, self-improvement, and chain-of-thought prompting. Each level includes the previous ones, following the Self-Refine cycle (Madaan et al., 2023).

**Hierarchy** — two configurations: a monolithic ReAct Planner versus a three-agent hierarchy (Planner, Analyst, ActionChooser) with strict JSON I/O contracts.

The Analyst assesses individual hosts; the ActionChooser ranks action candidates.

Sub-agents advise; only the Planner acts.

To compare across axes, the authors define **RPTS** — returns per token spent:

$$\text{RPTS} = \frac{G_{\text{config}} - G_{\text{obs}}}{\text{KTok}_{\text{config}}}$$

where $G_{\text{obs}}$ is the mean return under raw observations alone and KTok is total tokens in thousands. Negative values mean you would have been better off just showing the agent raw data.

## What You See Beats How Deeply You Think

Programmatic state abstraction is the cheapest lever with the biggest payoff.

Adding a deterministic network-status layer to raw observations (`obs` → `obs+net`) improves Llama's mean return by 76% and Qwen's by 71%.

No additional LLM calls. No new reasoning.

Just a Python function that compares process tables against a saved baseline and emits a compact JSON list of non-baseline hosts annotated with status and recency.

![Mean episode return across six context configurations for each model, showing that the programmatic network-status layer consistently lifts performance over raw observations]({{ "/assets/figures/1911d5765062/fig_004_01.png" | relative_url }})

The catastrophic failure rate tells the same story, sharply.

Under raw observations, Llama, Qwen, and G2.5FL fail catastrophically 96–98% of the time — essentially sleeping.

Under `obs+net` or `hist+net`, that rate drops below 10% for the strongest models.

![Catastrophic failure rate across context configurations, showing programmatic state tracking reduces failure from over 90% to under 10% for the best models]({{ "/assets/figures/1911d5765062/fig_014_06.png" | relative_url }})

Less-but-structured often beats more-but-unstructured.

The `hist+net` configuration (structured state plus compressed history, no raw observations) matches or outperforms the full-information `obs+hist+net` for four of six models.

Raw observations dilute the signal — removing them improves Llama's return by 44%.

This extends what Huang et al. (2024) showed for monolithic LLMs: self-correction without external feedback fails.

The fix is not better reasoning. It is better input.

## When Deliberation Cascades

Here is the destructive interaction.

Hierarchy without deliberation tools (`hier-base`) achieves the best absolute performance for four of six models. G3FP reaches a mean return of −16.1, approaching the top published deep reinforcement learning result of −3.47. Grok hits −24.0.

The benefit comes entirely from task decomposition — bounded assessments from the Analyst, ranked candidates from the ActionChooser, strict JSON contracts — not from deeper thinking.

Then the authors switch deliberation on for all three agents. Every result gets worse.

`hier-delib` degrades mean return relative to `hier-base` for all six models — up to 3.37× worse for Devstral (−37.8 → −127.4). Token costs balloon by 1.8–2.7×.

This is not a token-overhead story: `hier-base` on Grok uses roughly the same tokens as monolithic `+COT` (141.9K vs. 144.5K) yet achieves −24.0 versus −42.9. The loss is structural.

![Mean episode return comparing flat-base, hier-base, and hier-delib configurations across six models, showing hierarchy without deliberation outperforms hierarchy with distributed deliberation]({{ "/assets/figures/1911d5765062/fig_007_04.png" | relative_url }})

The authors call the mechanism a **deliberation cascade**. Independent critique loops in sub-agents amplify uncertainty without an arbitration protocol.

When the ActionChooser questions and critiques its own ranking, it introduces excessive qualifications — "Monitor (0.9), Analyse (0.8), Restore (0.7)."

The Planner cannot distinguish these hedged confidence labels from genuine warnings about the environment. Caution accumulates down the hierarchy.

The agent defers action until evidence becomes overwhelming, which in a failure-mitigation regime means costly delays.

The pattern echoes what Kim et al. found about multi-agent topology scaling: wiring alone does not tell the full story. Here, whether each node deliberates determines whether decomposition helps or hurts.

## Reading the Pareto Frontier

Pareto analysis across all twelve configurations lands on a consistent ordering.

1. **Context engineering** dominates the efficient frontier. `obs+net` achieves the highest RPTS for four of six models.

2. **Bounded hierarchy** sets the performance ceiling where it works. Four of six models improve substantially over their monolithic best.

3. **Monolithic deliberation** is capability-conditional — it helps some models and destabilizes others.

4. **Distributed deliberation** is dominated everywhere.

The design principle is a sequence, not a menu.

Invest in deterministic context infrastructure first, add bounded hierarchy second, treat deliberation as a costly option that should never be distributed across sub-agents by default.

## Where the Ground Shifts

Qualitative effects replicate across all six models — context helps, distributed deliberation hurts. But magnitudes vary by 2–10× and some effects reverse entirely.

Llama is harmed by hierarchy (−57.1 → −69.5). A Llama-only study would conclude hierarchy hurts; a Grok-only study would assert it is essential.

G2.5FL fails catastrophically across every configuration, suggesting a capability floor below which structured I/O contracts break down.

The scope constraints matter.

This is one environment — a fixed 13-host topology with a scripted adversary over 30 steps.

One fixed three-agent hierarchy.

Deliberation tools are tested in cumulative activation order only.

The models are mid-tier; frontier-scale models may respond differently.

Knowledge-free prompts cannot fully disentangle pretrained priors from architectural effects.

The cascade mechanism itself — sub-agents amplifying hedged qualifications that the Planner cannot distinguish from environmental warnings — deserves direct study.

How much does confidence gating help? Can explicit uncertainty arbitration protocols prevent it?

The pattern likely extends beyond cyber defense to any partially observable setting where agents must act under ambiguity.

## The Cheap Lever Wins

In adversarial settings where errors compound, the highest-return investment is not more reasoning.

It is cleaner perception.
