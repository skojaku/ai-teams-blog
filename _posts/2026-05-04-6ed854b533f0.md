*TL;DR: Agent Capsules makes it safe to merge multiple LLM agents into a single call by gating every mode switch on measured output quality—turning a manual cost–quality tradeoff into a runtime optimization that matches an omniscient oracle on every tested cell.*

## The Compound Execution Problem

A five-agent due-diligence pipeline running a thousand times a day fires at least a thousand LLM calls before lunch.

Every call pays the full overhead of system prompt ingestion, context setup, and per-request scheduling.

The obvious fix—merge those agents into one call—slashes tokens. It also silently breaks things.

Tool-using agents lose access to their tools. Smaller models compress five agent instructions into one shallow summary instead of producing per-agent depth.

No existing framework tells you when merging is safe. CrewAI, LangGraph, Google ADK—they all treat each agent as an independent dispatch unit.

The cost–quality tradeoff lands in your lap. Agent Capsules (AC) takes it off your desk.

## A Behavioral Fingerprint for When to Merge

AC does not ask whether a model is "smart enough" for compound execution. It asks how the model *behaves* in fine-grained mode—how much coordination overhead it generates, how many tools it calls, how deep its dependency chains run.

These four signals collapse into a single composition score:

$$s = 0.45\,r_{\text{oh}} + 0.25\,\min\!\left(\frac{n}{4}, 1\right) + 0.25\,\min\!\left(\frac{\bar{t}}{3}, 1\right) - 0.05\,\min\!\left(\frac{d}{\max(n{-}1,\,1)}, 1\right)$$

where $r_{\text{oh}}$ is the ratio of coordination overhead tokens to total output, $n$ is agent count, $\bar{t}$ is mean tool calls per agent, and $d$ is dependency depth.

The score is a behavioral fingerprint, not a capability ranking. Gemini-2.5-pro scores 0.177–0.180—statistically identical to GPT-4o-mini—because both invoke exactly one tool per agent. Sonnet and Haiku score above the 0.23 threshold precisely because they call 2.0–2.5 tools per agent.

When the rolling composition score clears `compose_at`, the controller fires.

When it doesn't, the group stays in fine mode. No per-model knobs.

## The Escalation Ladder

Merging is not a binary switch. AC runs three compound strategies in order.

**Standard compound.** One merged call, all agents batched. Saves 35–87% of input tokens. Breaks when tools get lost or smaller models compress the prompt.

**Two-phase compound.** Phase A runs per-agent tool-gathering calls; Phase B merges just the reasoning. Tool access stays intact. Saves 16–72%.

**Sequential compound.** Agent $k$ gets the outputs of agents $1, \dots, k{-}1$ as context, but retains its own dedicated call. No merging—each agent gets its own generation step. Saves 0–8% without output guidance.

The controller escalates automatically: `standard → two-phase → sequential`. Each tier fails the quality gate, and the controller upgrades.

The alternative—reverting to fine mode—is what happens when even sequential can't clear the floor.

The key insight: quality failure under compound is not a signal to stop batching. It is a signal to batch smarter.

## The Negative Result That Makes the Ladder Work

The obvious alternative to escalation is to give the merged call more context so the model can reason harder before compressing. The paper tested this directly.

A richer Phase A reasoning pre-pass improved analysis quality by +0.084 on Sonnet—but regressed research quality from 0.742 to 0.675. Injecting more context into a merged call gives the model more material to compress, not less reason to compress.

Sequential execution, with no merging, cleared both groups (research 0.758–0.833, analysis 0.775–0.783). The merged call itself is the compression bottleneck.

The escalation ladder encodes this: it recovers quality by un-merging, not by rewriting prompts.

## The Quality Gate

Every compound run gets shadow-evaluated against a fine-grained baseline by an LLM judge.

The gate uses a rolling mean over the last 10 runs—not a point estimate—to damp judge noise (measured standard deviation: 0.012–0.032).

If rolling quality drops below the floor, the controller reverts.

$$\text{revert} \iff \bar{q} < q_{\text{floor}}$$

On Sonnet, per-group mode selection clears `quality_floor = 0.75` for all three task types: research (standard, 0.775), analysis (sequential, 0.783), synthesis (standard, 0.833). No group requires fine mode.

On Haiku, the gate routes every group to fine exclusively. Haiku suppresses tool use entirely under compound framing—its behavioral stability index (tool calls in compound / fine) is 0.0.

The controller matches the oracle on every measured (model, group, mode) cell.

It routes compound exactly when the oracle would, and reverts to fine whenever quality would fail the floor. Zero per-model configuration.

## Against the Baselines

The head-to-head numbers are sharp.

Against a hand-tuned 14-agent LangGraph competitive-intelligence pipeline, AC uses 51% fewer fine-mode input tokens and 42% fewer compound-mode input tokens—at +0.020 and +0.017 quality respectively.

AC's fine-grained mode already beats LangGraph before compound even fires, thanks to automatic policy resolution, cache-aligned prompts, topology-aware context injection, and concise-output guidance.

Against DSPy's 5-agent due-diligence pipeline, AC uses 19% fewer tokens than uncompiled DSPy at quality parity—and 68% fewer tokens than MIPROv2 at +0.052 quality.

DSPy optimizes prompts at compile time; AC optimizes execution structure at runtime. The two approaches are composable, but runtime adaptation alone already beats compile-time optimization here.

The escalation ladder on a 7-agent code-review group lifts quality from 0.313 to 0.724 (+0.411) while cutting tokens by 10% and latency by 15%. Standard compound was structurally broken on this group—the `[AGENT:]` framing destroyed tool behavior—so escalation to two-phase was the only path to recovery.

![Escalation ladder validation on the 7-agent code-review group, showing quality recovery from 0.313 to 0.724 with token and latency savings]({{ "/assets/figures/6ed854b533f0/fig_014_07.png" | relative_url }})

Auto output guidance delivers 63–64% output-token savings on Sonnet and Haiku while staying quality-neutral. On already-terse Gemini-flash, it stays neutral rather than forcing harmful compression.

The selector applies concise only when mean fine-mode output exceeds 1,500 tokens per agent. No deployer is worse off under auto.

## The Honest Limits

The evaluation is primarily on Sonnet and Haiku. Haiku fails all compound modes because it suppresses tool behavior under merged framing—the entire approach cannot help models that lose tool behavior when prompts are restructured.

Sequential compound's token savings are minimal without output guidance (0–8%). The composition score thresholds were derived via grid search on two pipelines; generalization to very different pipeline topologies is unproven.

The approach is the first to treat multi-agent execution as a group-level optimization with empirical quality constraints. Whether the thresholds transfer is the open question—and it matters, because the thresholds are what make the oracle-equivalent routing work.

The cost–quality tradeoff is no longer yours to make.
